## 아이디어 · 주차 정확도 등급

주차존 안에 "어느 정도" 들어오면 확정 가능하되, **얼마나 잘 댔는지**를 B / A / S / SS로 채점한다.

### 확정 조건 완화

- 지금은 비행기 AABB가 주차존 AABB에 **완전 포함**(`AABB.encloses`)돼야 확정 대기.
- 이걸 "면적의 일정 비율(예: 70%) 이상 겹침"으로 완화해 애매하게 걸쳐도 확정은 가능하게 한다.
  판정 위치: [aircraft_collision.gd](../src/gameplay/aircraft/aircraft_collision.gd) `is_fully_parked()`.

### 채점 두 축 (확정 순간 스냅샷)

1. **위치 오차** — 비행기 중심과 주차존 중심의 XZ 거리.
2. **각도 오차** — 비행기 heading과 주차존 정면 방향(+Z 등)의 각도 차.

두 오차를 각각 0~1로 정규화해 합산(가중 평균)한 점수로 등급을 매긴다. 임계값은 인게임 튜닝.

| 등급 | 느낌 | 대략 기준 |
|---|---|---|
| B | 애매하게 들어옴 | 겹침 조건만 통과 |
| A | 어느 정도 정렬 | 위치·각도 오차 중간 이하 |
| S | 거의 완벽 | 중앙 근처 + 잘 정렬 |
| SS | 완벽하게 주차 | 중앙 + 정면 정렬 오차 최소 |

### 후속 / 열린 질문

- 등급은 확정(스페이스) 순간에만 계산 → 성공 HUD에 표시.
- 각도 오차는 AABB가 회전을 감싸는 박스로 뭉개므로, 비행기 실제 `global_rotation.y`를 직접 읽는 게 정확.
- SS 연출(파티클/효과음)과 연동, 점수 누적/기록 여부는 추후.
